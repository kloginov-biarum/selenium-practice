CSS Selectors Practice

Write reliable CSS locators for the elements below.

1. Attribute selectors

Locate inputs by their attributes:

input[type="email"] — by exact attribute
input[name^="user"] — name starts with "user"
input[name$="word"] — name ends with "word"
input[name*="ail"] — name contains "ail"
input[disabled] — has disabled attribute

4. When CSS vs XPath
Use caseBest tool
By id, class, attributeCSS (faster, shorter)
nth-child / first / lastCSS
Adjacent / general siblingCSS
Match by visible textXPath (text(), contains())
Walk up the tree (parent / ancestor)XPath
Match exact text contentXPath
Combine multiple conditions on one nodeBoth work, XPath more flexible
5. Reliable locator rules
  1. Prefer id if present and stable
  2. Use data-* attributes when available — they don't change for visual reasons
  3. Avoid absolute paths (/html/body/div/...)
  4. Avoid auto-generated classes (css-1a2b3c)
  5. Avoid coupling to text in i18n apps (text changes per language)
  6. Prefer contains() / starts-with() over exact match for dynamic ids
  7. Test your locator in DevTools first: $x("//...") or $$("css")
