XPath Practice

Write reliable XPath locators for the elements below.

1. Axes — parent / child / following-sibling

Find the Buy button next to the product named “Wireless Mouse”.

#ProductPriceAction
1 Mechanical Keyboard $120
2 Wireless Mouse $45
3 USB-C Cable $15
Hint: use following-sibling to jump from the product cell to the action cell.
Example: //td[text()='Wireless Mouse']/following-sibling::td/button
2. Predicates — contains() / starts-with()

Each card has a dynamic id that starts with card_ followed by random characters. Find the card whose title contains the word “Premium”.

Standard Plan

Basic features for everyone.

Premium Plan

Advanced features for power users.

Enterprise Plan

Custom solutions.

Hint: //div[starts-with(@id,'card_') and .//h5[contains(text(),'Premium')]]
3. text() — exact text matching

Find the link with the exact text “Read more” (not “Read more about us” or “Read”).

  • Read
  • Read more
  • Read more about us
  • Read carefully
Hint: //a[text()='Read more']
4. parent — go up the tree

The error icon below has no useful attributes. Find its parent <li> to read the message.

  • Saved successfully ✔
  • Network error ✘
  • Loading... ⏳
Hint: //span[@class='icon err']/parent::li
5. Absolute vs relative — why relative wins

Locate the highlighted button. Try both:

  1. Absolute: /html/body/div/div[5]/div/button — fragile, breaks if structure changes.
  2. Relative: //button[@id='target-btn'] — robust.
